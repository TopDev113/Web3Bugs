Deleted User
Hi, I'm curious why a while loop is needed when performing _searchOrderbook in the _updateBidInOrderbook function when the _prevUser parameter should already tell you where to insert the new bid? (edited)
Splidge | Reality Cards 🎴 23-Aug-21 02:22 AM
It could be that multiple users place bids at the same (or similar) times. Lets say that Alice is the current owner, Bob and Carol both place bids at the same time, Both Bob and Carol would pass the starting location "Alice", if we didn't search at all then it would depend on the order of processing if Bob or Carol succeeds, either-way one of them would fail. By allowing searching we remove some of the issues around transaction processing times. We have considered not needing the starting location at all, in a previous version it was so gas intensive to search the orderbook the starting location was required to be within 10 positions otherwise it would fail. However now we can search 1000+ positions it's probably not necessary to do this anymore (a bid that far down the orderbook would be very unlikely to ever be used), we simply haven't removed it because we can't be sure if this will be required or not in the future.

Deleted User
I also want to check re. the linked list used in RCOrderBook.sol. Is the further you are in the linked list, the higher your price? or is it the reverse?
Splidge | Reality Cards 🎴 23-Aug-21 02:39 AM
Starting with the market record the .next bid is the current owner of the card, the highest priced bid (if it exists, otherwise it points back at the market record). Then going .next each time the price of the bids decreases eventually arriving back at the market record. So the further you are down the list the lower your price. I suppose a version could be made where the list might not need to be doubly linked and .prev could be removed, although we'll not be making that change before launch, maybe for V2. The Leaderboard is very similar, with the first record being the user with the longest time held, then the .next user has less time held etc etc. I did accidentally have this backwards for a while but it didn't matter because the position in the Leaderboard doesn't matter, only if you're on it or not.
