Justin Goro | Behodler 30-Jan-22 07:10 PM
PSA Hi wardens. One issue I see coming up a lot is something that appears as a vulnerability in a vacuum but in the correct context isn't. Limbo is basically open to admin manipulation until the conigured bool is set to true. The purpose of this is so that when I deploy Limbo initially, I can configure it quickly. If this wasn't in place, I'd have to issue proposals and vote on them just to set up the contract. Once configured is set to true, the admin powers are permanently removed. The pre configured state will be during a time where I'm the only person who knows the contract exists. And since there are no value flows, there will be no detection by mining bots. (edited)
19:12
On a similar note, with governance, the proposals which cause havoc to their target contract are not considered threats since the community can simply vote them down. And we might have some perculiar reason to want such proposals. Perhaps it's the only way to thwart a more dangerous exploit. (edited)
